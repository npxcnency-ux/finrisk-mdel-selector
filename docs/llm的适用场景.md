这个问题触及 LLM 的架构本质。四个现象背后归结为三个根因，且**全部是通用问题**。

---

## 三个根因

### 根因 A：训练目标——预测下一个最可能的 token

所有主流 LLM 的预训练目标是**最小化交叉熵损失**：给定前文，预测下一个 token 的概率分布。

这意味着模型学到的是**训练语料中的条件频率分布**，不是「什么是正确答案」。

```
P("XGBoost" | "表格数据异常检测推荐") >> P("先用 Isolation Forest" | 同上)
```

不是模型不知道 IF，是生成时概率路径把它压下去了。这直接导致了**现象 1（高频路径压制）**。

**关键区分**：模型的知识（权重中编码的信息）和模型的输出（解码时走的概率路径）是两回事。知识是有的，但解码策略选择了高概率路径。

---

### 根因 B：生成机制——自回归、左到右、不可回溯

Transformer 生成文本是逐 token 从左到右的。一旦输出了「推荐 XGBoost」，后续所有 token 都以此为条件——模型没有内建的「停下来重新审视前面输出是否合理」的机制。

```
token 1 → token 2 → token 3 → ... → token N
         ↑ 每一步都依赖前面所有 token
         ↑ 没有回溯、没有全局规划
```

这直接导致了：
- **现象 2（满足即停）**：找到一条看起来通的路就一直走下去，不会自发回头检查
- **现象 4（约束丢弃）**：前面提到了约束 A，生成到中段时注意力权重已经分散，约束 B 被隐性降权

人脑的 System 2 可以暂停、回顾、重新评估。自回归生成没有这个环节——除非外部机制（如 skill 的门控结构）强制插入检查点。

---

### 根因 C：奖励信号——RLHF 优化的是「看起来有帮助」

RLHF（人类反馈强化学习）阶段，标注者倾向于给「完整、流畅、自信」的回答高分，给「我需要更多信息」低分。

模型学到的隐性策略：

```
给出完整回答 → 高奖励
承认不确定性 → 低奖励
主动提问 → 更低奖励（用户可能觉得你没用）
```

这直接导致了**现象 3（假设而非提问）**。模型填补假设不是因为它判断假设合理，而是因为「给出完整答案」这个行为本身在训练中被强化了。

---

## 四个现象的根因映射

| 现象 | 根因 A（训练目标） | 根因 B（生成机制） | 根因 C（奖励信号） |
|------|:-:|:-:|:-:|
| 1. 高频路径压制 | **主因** | 加剧 | — |
| 2. 满足即停 | — | **主因** | 加剧 |
| 3. 假设而非提问 | — | — | **主因** |
| 4. 约束冲突丢弃 | 加剧 | **主因** | — |

---

## 是否通用？

**是，100% 通用。** 因为三个根因来自架构和训练范式本身，不是某个模型或某个领域的特有问题。

但**严重程度因场景而异**：

| 场景特征 | 严重程度 | 原因 |
|---------|---------|------|
| 正确答案是高频答案 | 低 | 概率路径恰好是对的 |
| 正确答案反直觉 | **高** | 概率路径和正确路径相反 |
| 单一约束 | 低 | 不存在约束冲突 |
| 多约束并存且冲突 | **高** | 注意力分散 + 无约束求解器 |
| 信息完整 | 低 | 无需假设 |
| 信息残缺 | **高** | 假设填补被强化 |
| 领域在训练数据中占比大 | 中 | 高频路径更强，但知识也更丰富 |
| 领域在训练数据中稀疏 | **高** | 知识不足 + 高频路径来自相邻领域的迁移 |

**财务风控恰好集齐了所有高严重度条件**：正确答案经常反直觉、多约束并存、信息通常不完整、领域在通用训练数据中相对稀疏。所以这个 skill 的价值在这个领域特别显著。

但同样的问题在医疗诊断、法律推理、工程安全分析等领域同样存在——任何「正确答案不是最常见答案」的领域都会遇到。

---

## 反过来想：什么场景下这些问题不严重？

- 写常见代码（React 组件、CRUD API）→ 高频路径就是对的
- 信息完整的翻译/摘要 → 无需假设
- 单一明确目标的任务 → 无约束冲突
- 通用知识问答 → 训练数据覆盖充分

这也解释了为什么很多人觉得"LLM 已经够好了"——在他们的使用场景里，这些结构性缺陷恰好不被触发。
