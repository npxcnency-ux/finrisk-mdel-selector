# 算法推荐参考矩阵

## 有标签路径

| 条件 | 推荐算法 | 原因 |
|------|---------|------|
| 样本 < 5000 + 强解释性（监管场景）| 评分卡（WOE/IV + 逻辑回归）| 系数可直接出具书面解释，满足银行/保险监管要求 |
| 样本 < 5000 + 规则需求 | 决策树（深度≤5） | 可导出 if-else 规则 |
| 样本 > 5000 + 中等解释性 | XGBoost + SHAP | 效果强，SHAP 辅助解释 |
| 样本 > 5000 + 高解释性 | LightGBM + 规则提取 | 速度快，可配合规则层 |
| 样本 > 10 万 + 中等解释性 | GBDT + LR 两阶段 | 树做特征交叉，LR 做可解释输出，概率校准更好 |
| 时序消费行为 | LSTM + 注意力机制 | 捕捉序列中的异常模式 |
| **严重不平衡（< 1%）** | **Isolation Forest 初筛 → XGBoost 精排** | **即使有标签也要先扩召回，再精排** |

## 弱标签/半监督路径

适用场景：有少量经确认标签（< 500 条），或标签质量存疑（"疑似"未经二次核实）。

| 条件 | 推荐方法 | 原因 |
|------|---------|------|
| 有少量确认标签（< 500 条）| Label Propagation / Label Spreading | 利用有标签样本向无标签样本传播，适合特征空间连续的场景 |
| 标签质量不稳定 | Self-Training（自训练）| 用高置信度预测结果扩充标签集，迭代训练 |
| 有标签 + 无标签数据量大 | Semi-Supervised Isolation Forest | 将已知异常作为约束引导无监督初筛方向 |
| 先无监督、后有监督 | **Isolation Forest 初筛 → 人工审核 → XGBoost 精排** | 冷启动最实用路径，无需等到足够标签再启动 |

**注意**：弱标签场景往往比直接走无监督更容易被忽视。如果用户说"有几百条疑似记录但未经确认"，不要直接上 XGBoost，先评估标签质量。

## 无/弱标签路径（完全无标注）

| 条件 | 推荐算法 | 原因 |
|------|-----------|------|
| 单维异常（金额/频次）| IQR / Z-Score 规则 | 可解释，易维护 |
| 多维异常 + 需要解释 | K-Means 分群 → 离群分析 | 结果可对业务解释 |
| 多维异常 + 效果优先 | Isolation Forest | 财务场景最常用，快速高效 |
| 局部密度异常（拆单等）| LOF（Local Outlier Factor）| 对拆单、频繁小额敏感 |
| 特征高维或有交叉关系 | Autoencoder（重构误差）| 学习正常模式，偏差即异常 |
| 时序消费 + 无标签 | LSTM Autoencoder | 捕捉时序偏离正常序列 |

## 标签不平衡处理策略

| 异常比例 | 判定 | 处理策略 |
|---------|------|---------|
| > 20% | 平衡 | 无需特殊处理 |
| 5%–20% | 轻度不平衡 | 调整 `class_weight='balanced'` |
| 1%–5% | 中度不平衡 | SMOTE 过采样 + class_weight |
| < 1% | 严重不平衡 | Isolation Forest 初筛 → 监督精排 |

## 评估指标选择

| 场景 | 主要指标 | 原因 |
|------|---------|------|
| 极不平衡（< 1%）| PR-AUC | ROC-AUC 在此场景虚高，全判正常也能 0.9+ |
| 模型选型对比 | AUC-ROC | 不受阈值影响，反映整体区分能力 |
| 运营阈值决策 | Precision@K | K = 团队每日处理量，关注前 K 条命中率 |
| 合规漏报代价大 | Recall | 设定最低召回率底线（如 80%） |
| 综合评估 | F1 | Precision 和 Recall 的调和平均 |

**绝对不要用 Accuracy**：异常率 0.5% 时，全判正常也有 99.5% 准确率。
